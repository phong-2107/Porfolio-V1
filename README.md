## Personal Direction

This portfolio represents the point where my backend and full-stack background meets a more visual frontend direction.

My work started around practical systems: REST APIs, authentication, payment integration, database-backed applications, cloud deployment, and team projects. With this website, I wanted the interface itself to communicate that same technical foundation, but through a more memorable visual language.

The result is a portfolio that presents:

- My full-stack project history.
- My frontend and UI engineering direction.
- My interest in motion, 3D scenes, and scroll storytelling.
- My preference for structured, reusable, production-ready code.
- A personal visual identity built around dark depth, orange energy, and cyan technical detail.

## Experience Flow

The page is structured like a short interactive story:

```txt
Hero / Identity
  -> Technical Profile
  -> Personal Bio
  -> Visual Showreel
  -> Career Timeline
  -> Selected Projects
  -> Contact
```

Each section has a specific role. The hero creates the first impression, the technical profile explains the stack, the project cards prove the work, and the contact section closes the experience with a direct path to connect.

## Motion Architecture

The motion system is intentionally separated from content so animation can be improved without rewriting the whole page.

| File | Responsibility |
| --- | --- |
| `src/lib/gsap.ts` | Central GSAP plugin registration. |
| `src/components/utilities/SmoothScrollProvider.tsx` | Lenis setup and ScrollTrigger synchronization. |
| `src/components/motion/StoryMotionProvider.tsx` | Shared section reveal behavior. |
| `src/hooks/useMagneticElements.ts` | Magnetic hover physics for interactive elements. |
| `src/components/animated/MotionCursor.tsx` | Custom cursor layer. |
| `src/components/hero/SplineHeroScene.tsx` | Spline scene integration and scroll-linked 3D behavior. |

The implementation favors `transform`, `opacity`, and subtle `filter` changes so the motion feels smooth while staying performance-aware.

## Visual Language

The design avoids the typical bright portfolio template look. It is built around a darker, more cinematic system:

- **Deep base:** creates contrast and depth.
- **Orange accent:** marks primary actions and creative energy.
- **Cyan support:** reinforces technical metadata and HUD-style details.
- **Large editorial type:** gives the page confidence and scale.
- **Glass surfaces and fine borders:** keep the UI detailed without becoming noisy.
- **3D composition:** makes the hero feel like a scene instead of a flat banner.

## Folder Structure

```bash
src/
|-- app/
|   |-- providers/        # App-level provider shell
|   `-- routes/           # Route composition
|-- components/
|   |-- animated/         # Motion cursor and animation-only UI
|   |-- header/           # Header, menu, and sidebar navigation
|   |-- hero/             # Spline hero scene and hero visual wrappers
|   |-- layout/           # Page shell, app wrapper, loading screen
|   |-- motion/           # Route-scoped story motion provider
|   |-- sections/         # Hero, profile, about, showreel, career, projects, contact
|   |-- social/           # Social link components
|   |-- ui/               # Cards, badges, modals, buttons, HUD components
|   `-- utilities/        # Smooth scroll, text animation, navigation helpers
|-- data/                 # Portfolio content, skills, projects, process, story chapters
|-- hooks/                # Loading, pointer, magnetic, and motion hooks
|-- lib/                  # GSAP registration, motion helpers, navigation constants
|-- styles/               # Tokens, base styles, component styles, motion, responsive CSS
|-- css/                  # Legacy and imported CSS assets
|-- types/                # Shared TypeScript types
|-- App.tsx
|-- main.tsx
`-- index.css
```
